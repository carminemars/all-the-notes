# gRPC Documentation

Created by **Google** in 2016, gRPC is a high-performance, open-source framework for building remote procedure call (RPC) APIs. It uses HTTP/2 for transport, Protocol Buffers as the interface description language, and provides features such as authentication, load balancing, and more.

Let's take a step back and let's talk about RPC in general. RPC is a powerful paradigm for building distributed systems, allowing clients to call methods on a server as if they were local. gRPC takes this concept to the next level by providing a modern, efficient, and scalable framework for building RPC APIs.
