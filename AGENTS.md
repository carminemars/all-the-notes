# AGENTS.md — all-the-notes

Personal notes site built with [Docusaurus 3.10](https://docusaurus.io/), React 19, TypeScript 6.0.

## Commands

```bash
npm install          # install deps (package-lock.json — not yarn despite README)
npm run start        # dev server with HMR
npm run build        # static build → build/
npm run typecheck    # tsc (runs separately, not part of start/build)
npm run serve        # serve production build locally
npm run clear        # wipe .docusaurus cache
```

## Package manager

Uses **npm** (has `package-lock.json`). The `yarn` commands in `README.md` are stale Docusaurus defaults — ignore them.

## Build & CI quirks

- `docusaurus.config.ts` has `onBrokenLinks: 'throw'` — broken internal links fail the build.
- Build artifacts: `build/` and `.docusaurus/` are gitignored.
- `tsconfig.json` is **not** used by `docusaurus start/build`. It only exists for IDE autocompletion and `npm run typecheck`.
- If you move/rename pages, run `npm run clear` first to avoid stale cache.

## Theme & customizations

Swizzled components live in `src/theme/`:
- `src/theme/Layout/index.tsx` — injects GSAP + mouse-follower custom cursor
- `src/theme/Footer/index.tsx` — custom footer with GitHub link

Homepage at `src/pages/index.tsx` — bouncing avatar with DVD-logo-style animation, glitch easter egg on click.

## Code conventions

- Orange color scheme via CSS custom properties in `src/css/custom.css`.
- Dark mode only (`disableSwitch: true`, `defaultMode: 'dark'`).
- No linter or formatter config present — none enforced.
- No test framework — none expected.

## Content

- **Docs** (`docs/`): autogenerated sidebar (`sidebars.ts`), MDX files, `_category_.json` for taxonomy.
- **Blog** (`blog/`): MDX posts, authors in `authors.yml`, tags in `tags.yml`.
- **Images** in `static/img/`.
